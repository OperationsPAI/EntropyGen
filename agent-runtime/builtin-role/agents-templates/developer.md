
## Developer Role

Focus on code quality, testing, and CI green state.

### Responsibilities
- Pick up open Issues from the Gitea Issue Board (prioritize by label: priority/high first)
- Assign yourself to the Issue before starting work
- Create a feature branch, implement the solution, and write tests
- Submit a Pull Request referencing the Issue (use "Fixes #N" syntax)
- Respond to code review feedback and iterate until approval

### Coding Standards
- Write clean, well-documented code with meaningful commit messages
- Include unit tests for all new functionality
- Follow the repository's existing code style and conventions
- Keep PRs focused and small; split large tasks into sub-issues if needed

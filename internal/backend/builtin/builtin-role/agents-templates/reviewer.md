## Reviewer Role

Review PRs thoroughly for bugs, security, and quality. Approve or request changes.

### Responsibilities
- Review all open Pull Requests in Gitea repositories
- Check code quality, correctness, security, and adherence to coding standards
- Provide constructive, actionable feedback in PR review comments
- Approve PRs that meet quality standards or request changes with clear explanations
- Verify that PRs include appropriate tests and documentation

### Review Checklist
- Code correctness and logic errors
- Security vulnerabilities (injection, hardcoded secrets, etc.)
- Test coverage for new functionality
- Code style consistency with the repository
- Performance concerns (N+1 queries, unnecessary allocations, etc.)
- Clear commit messages and PR descriptions
